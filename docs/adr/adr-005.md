# ADR 005 : Scopes vs Flags pour les Variantes Régionales

**Statut :** Validé (Décision de design v2.4.0)  
**Contexte :** Nécessité de gérer la mobilité de l'Épiphanie (6 janv. vs Dimanche entre le 2 et le 8 janv.) et son impact sur le Baptême du Seigneur.

**Décisions :**

1. **Rejet des Flags de Compilation :** Le concept de "Control Plane" via flags globaux (ex: `--region-fr`) est écarté pour maintenir la Forge comme une fonction pure : `(SourceFiles) -> Binary`.
2. **Primauté du Système de Scopes :** La différenciation algorithmique est déportée dans le **Data Plane**. Une variante régionale est traitée par une surcharge de fichier dans la hiérarchie de répertoires : `universale/` < `nationale/{id}/` < `diocesana/{id}/`.
3. **Ancres Virtuelles :** Si une région change la règle de calcul d'une ancre (ex: `epiphania`), le fichier YAML correspondant dans le scope national redéfinit l'ancre. Les fêtes dépendantes (ex: `in_baptismate_domini`) héritent naturellement de la nouvelle position via le graphe de dépendances de l'Étape 3.

**Implication AOT :** Le binaire `.kald` ne contient aucune logique conditionnelle. La "vérité" du calendrier est figée par le choix des sources YAML ingérées au début du pipeline.
